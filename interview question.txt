Virtual DOM

Virtual DOM is the lighttweigtht representation of the Real DOM when object state changes virtual DOM will 
change only that object rather than updating all the object.

Working of Virtual DOM
Here's a breakdown of how the Virtual DOM works:
Rendering the Virtual DOM: React creates a virtual representation of the UI as a tree of JavaScript objects.
Updating State: It generates a new virtual DOM tree to reflect the updated state when the application state changes.
Diffing Algorithm: React compares the new Virtual DOM tree with the previous one using its efficient diffing 
algorithm to identify the minimal set of changes required.
Updating the Real DOM: React applies only the necessary changes to the real DOM, optimizing rendering performance.

Real DOM

React Fiber is the core reconciliation engine introduced in React 16 to improve rendering performance and make UI updates smoother.

reconciliation - Reconciliation in React is the process through which React updates the 
DOM to match the virtual DOM. When a component's state or props change, React creates a
new virtual DOM tree and compares it with the previous one. This comparison process is 
called "diffing." React then updates only the parts of the actual DOM that have changed, 
making the updates efficient and fast.

down through the DOM tree to the target element.

Type coercion in JavaScript is the automatic or implicit conversion of values from one data type to another 
(e.g., number to string, or string to number) when an operation involves mismatched types. This is a core 
feature of JavaScript as a weakly-typed language, but it can lead to unexpected results if the rules are not understood. 

Types of Coercion

JavaScript uses two types of type conversion: 

Implicit Coercion (Automatic): The JavaScript engine automatically converts types during operations without the 
developer's direct intervention. This happens in arithmetic operations, logical operations, and loose comparisons
 (using ==).


Explicit Coercion (Type Casting): The developer intentionally converts one type to another using built-in functions
 or methods like Number(), String(), Boolean(), parseInt(), or .toString().

a shallow copy shares references to nested data, while a deep copy creates a completely independent replica of the entire structure. 

//shallow copy
const original = {
  name: 'Jane',
  work: { employer: 'Acme' }
};
const shallowCopy = { ...original };

shallowCopy.name = 'John'; // Top-level change: original.name is 'Jane'
shallowCopy.work.employer = 'Spectre'; // Nested change: original.work.employer is now 'Spectre' (shared reference)

console.log(original.work.employer); // Output: 'Spectre'

//Deep Copy

const original = {
  name: 'Jane',
  work: { employer: 'Acme' }
};
const deepCopy = JSON.parse(JSON.stringify(original)); // Or use structuredClone(original)

deepCopy.name = 'John';
deepCopy.work.employer = 'Spectre'; // Changes only affect deepCopy

console.log(original.work.employer); // Output: 'Acme' (original remains unchanged)

Object.entries() -- The Object.entries() method returns an array of the key/value pairs of an object.
const person = {
  name: "John",
  age: 30,
  city: "New York"
};

const entries = Object.entries(person);
console.log(entries);
// Output: [ ["name", "John"], ["age", 30], ["city", "New York"] ]

Object.key()-- The Object.keys() method returns an array with the keys of an object.

The Object.keys() method does not change the original object.

Object.freeze() -- The Object.freeze() static method freezes an object. Freezing an object 
prevents extensions and makes existing properties non-writable and non-configurable.
const obj = {
  prop: 42,
};

Object.freeze(obj);

obj.prop = 33;
// Throws an error in strict mode

console.log(obj.prop);
// Expected output: 42
-----------------------------------------------------------------------------------------
Object.seal() -- Object.seal() in JavaScript is a static method that locks an object, preventing 
new properties from being added and marking all existing properties as non-configurable.

const user = { name: 'Alice' };
Object.seal(user);

user.name = 'Bob'; // Allowed: Updating existing value
user.age = 25;     // Silently fails (or TypeError in strict mode)
delete user.name;  // Silently fails (or TypeError in strict mode)

console.log(user); // Output: { name: 'Bob' }
